# User-Facing Error and Guidance Messages

This catalogue defines clear Dashboard messages and practical recovery actions for users of the Hydroecology Toolkit Dashboard. Raw programming errors, local file paths, stack traces, package names, and console output must never be shown to ordinary users. Technical details may be recorded separately in internal developer logs.

Biology and Flow form the core analysis. Water Quality (WQ) and River Habitat Survey (RHS) data are optional enrichment, so an absent or unsuccessful WQ/RHS import must not imply that valid core Biology and Flow results have failed.

## 1. File upload and file format

| ID | Dashboard area / trigger | Message type | User-facing message | What it means | What to do next | Developer reference only |
|---|---|---|---|---|---|---|
| FILE-01 | No file has been selected | Information | No file has been selected yet. | The Dashboard is waiting for a file; nothing has failed. | Choose the appropriate file when you are ready, or continue without it if the data are optional. | Initial upload states in `server.R`; WQ/RHS/local upload helpers |
| FILE-02 | Uploaded file is no longer available | Error | The selected file could not be found after upload. | The upload did not reach the Dashboard successfully or is no longer available. | Select the file again and retry the upload. | `read_uploaded_csv_safely()`; `read_site_metadata_csv()` |
| FILE-03 | CSV cannot be read | Error | The file could not be read as a CSV file. | The file may use an unsupported format, separator, character encoding, or inconsistent rows. | Open the file in a spreadsheet application, save it as a UTF-8 comma-separated CSV, and upload it again. | Guarded `data.table::fread()` paths in `server.R` and `R/dashboard_backlog_helpers.R` |
| FILE-04 | File has headers but no data | Error | The file is empty. Add at least one data row and upload it again. | A header may be present, but there are no records to process. | Add the required records beneath the header and retry. | Empty-row checks in WQ, RHS, mapping, and local-file validators |
| FILE-05 | Unsupported file type | Error | This upload accepts CSV files only. | The selected file type is not supported by this upload control. | Export the data as CSV and upload the exported file. | `fileInput(..., accept = c(".csv", "text/csv"))` in `ui.R` |
| FILE-06 | Local Flow file contains additional columns | Warning | The Local Flow file was accepted, but additional columns were ignored. | Only the required Flow fields are used in the core Flow workflow. | Review the listed ignored columns and move source or mapping information to the site-mapping file where appropriate. | `validate_local_flow()` in `R/dashboard_backlog_helpers.R` |

## 2. Required columns and data validation

| ID | Dashboard area / trigger | Message type | User-facing message | What it means | What to do next | Developer reference only |
|---|---|---|---|---|---|---|
| DATA-01 | Site-mapping file is missing required columns | Error | The site-mapping file is missing one or more required columns. | The Dashboard cannot connect the required datasets with the current headers. | Compare the headers with the mapping template, add the columns named in the message, and upload the file again. | `validate_supporting_mapping()`; required mapping fields in `ui.R` |
| DATA-02 | Mapping contains duplicate column names | Error | The site-mapping file contains duplicate column headings. | The Dashboard cannot determine which copy of a repeated column to use. | Rename or remove duplicate columns so every heading is unique, then upload again. | `parse_site_metadata()` in `R/site_mapping_helpers.R` |
| DATA-03 | Local invertebrate file is missing required columns | Error | The local invertebrate file is missing required information. | One or more required site, date, taxon, or abundance fields are absent. | Use the current template and add the missing columns named in the message. | `validate_local_invertebrate()` |
| DATA-04 | Local Flow file is missing required columns | Error | The Local Flow file must contain site ID, date, and Flow value columns. | Flow processing cannot begin without all three fields. | Add the missing columns using the template headings and upload the file again. | `validate_local_flow()`; canonical fields `flow_site_id`, `date`, `flow` |
| DATA-05 | Required site ID is blank | Error | One or more required site IDs are blank. | Records without a required site ID cannot be mapped or processed reliably. | Fill in the missing site IDs, preserving any leading zeros, and upload the corrected file. | Blank identifier checks in local Flow and mapping helpers |
| DATA-06 | Date is blank or cannot be understood | Error | One or more dates are missing or are not in a recognised date format. | The affected records cannot be placed in the correct analysis period. | Correct the listed dates using a consistent calendar-date format and retry. | Date parsing in `validate_local_flow()` and `wq_rhs_parse_date()` |
| DATA-07 | Required numeric value is missing or invalid | Error | One or more required values are blank or are not valid numbers. | The affected records cannot be used in the requested calculation. | Correct the listed values, remove text from numeric fields, and upload the file again. | Flow numeric checks; WQ result standardisation; model complete-case checks |
| DATA-08 | WQ file is missing required fields | Error | The WQ file is missing information required to build the WQ summary. | The Dashboard cannot identify the monitoring site, sample date, determinand, result, or unit for every required record. | Compare the file with the WQ template, add the fields named in the message, and rebuild the summary. | `standardise_wq_contract_records()`; required WQ contract fields |
| DATA-09 | RHS file uses an unsupported identifier | Error | The RHS file must use the RHS survey ID column shown in the template. | The file uses a legacy or ambiguous RHS identifier that cannot be mapped safely. | Rename the source column to the required RHS survey ID heading and remove any legacy duplicate identifier column. | `normalise_rhs_records()`; `rhs_survey_id` contract |
| DATA-10 | Flow source value is not recognised | Error | The Flow source is not recognised. Use HDE or NRFA. | The site-mapping row names a source that the Dashboard does not support. | Correct the source value in the site-mapping file and validate it again. | `normalise_site_metadata_flow_input()` |

## 3. Site ID, date, and data mapping

| MAP-01 | Site metadata have not been supplied | Error | Add site-mapping information before importing or joining data. | The Dashboard does not yet know how the Biology, Flow, WQ, or RHS site identifiers relate. | Paste or upload the site-mapping data, review the preview, and then retry the requested step. | `parse_site_metadata()` and mapping-dependent server paths |

| MAP-04 | Biology samples begin before available Flow data | Warning | Some Biology samples occur before the available Flow record begins. | The core dataset can still be created, but early Biology samples may not have matching Flow information. | Review the listed sites and dates, import an earlier Flow period if available, or interpret the reduced coverage carefully. | Biology-versus-Flow start-date warning in `server.R` |

## 4. Biology and Flow core analysis

| INFO-03 | Flow source was left blank | Information | HDE was selected as the default Flow source. | No source was supplied for the mapped Flow site, so the Dashboard used its normal default. | Review the source selection and change it to NRFA only where an alternative source is required. | `normalise_site_metadata_flow_input()` provenance |
